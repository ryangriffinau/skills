# Agent Flywheel ÔÇö working cheat-sheet (human)

The day-to-day loop. A **core artifact layer** (Agent Mail + Beads) plus an optional **NTM swarm layer** for running many agents at once. **One-time install & config live in [`setup.md`](./setup.md)** ÔÇö do that first on a new machine or repo.

> **Golden rule: one shared working tree, one branch, no git worktrees.** Agents coordinate by reserving files through Agent Mail; worktrees defeat that and are never used.

---

## 1. Mental model

| Layer | Tools | Role |
|---|---|---|
| **Core** | **Agent Mail**, **`br`** (beads), **`bv`** (viewer) | file-lease coordination ┬Ě the task graph ┬Ě graph-aware triage |
| **Multipliers** | **NTM** (tmux room), **DCG** (destructive guard), **UBS** (bug scan), **CASS** (session memory) | speed, safety, quality, recall |
| **Human layer** | **Linear** | where features originate + ship-status lives (a one-way projection of beads) |

The core loop runs with just Agent Mail + `br` + `bv` and a couple of agents; **NTM is the room you open to run several agents at once.**

**Watch progress with bare `bv`** ÔÇö the interactive TUI of the whole graph. (`bv --robot-*` is JSON *for agents*, not for you; `br epic status` is a one-off text check.)

---

## 2. The workflow (6 stages) ÔÇö and what NTM covers

| Stage | What happens | NTM? |
|---|---|---|
| 1 ┬Ě **Plan** | brain-dump Ôćĺ sharpen Ôćĺ competing plans Ôćĺ synthesize Ôćĺ stress-test | only the optional parallel-draft step |
| 2 ┬Ě **Encode** | turn the final plan into a Beads graph | no (CLI) |
| 3ÔÇô5 ┬Ě **Triage / Implement / Close** | claim Ôćĺ reserve Ôćĺ code Ôćĺ test Ôćĺ review Ôćĺ close, looping | **yes ÔÇö the swarm** |
| 6 ┬Ě **Ship** | commit Ôćĺ PR Ôćĺ merge Ôćĺ changelog Ôćĺ close Linear | partly |

### Stage 1 ÔÇö Plan (highest-leverage; spend real effort here)
1. **Brain-dump Ôćĺ brief:** sharpen with **`/grill-me`** or **`/grill-with-docs`** (`/p-idea-wizard` if the feature isn't chosen yet). No template ÔÇö it's conversational.
2. **Pull prior context:** `cass pack --robot "<topic>"` ÔÇö reuse past decisions instead of re-deriving.
3. **Competing plans:** run **`/p-draft-plan`** in several strong models (GPT Pro, Opus, Gemini, Grok) ÔÇö or in parallel agents via `ntm spawn <name> -t parallel-explore`.
4. **Synthesize:** **`/p-synthesize-plans`** Ôćĺ one hybrid plan.
5. **Stress-test:** **`/p-reality-check`** + **`/p-pre-mortem`**. Save it in-repo (`docs/specs/<x>/plan.md`).

### Stage 2 ÔÇö Encode
**`/p-plan-to-beads`** Ôćĺ epics/tasks via `br`, with deps + self-contained comments. Verify: `bv` (the TUI), or `bv ready`.

### Stages 3ÔÇô5 ÔÇö the swarm
Each agent loops: `bv` Ôćĺ reserve files via Agent Mail Ôćĺ `cass pack` Ôćĺ implement + test Ôćĺ `ubs --staged --fail-on-warning` Ôćĺ **`/p-fresh-eyes-review`** Ôćĺ `br close` Ôćĺ **commit + push immediately** (┬ž4). Launch + monitor in ┬ž3.

## 3. Launch & monitor the swarm

**Drive the swarm with the `flywheel-conductor` skill from your own agent session**
(Claude Code or Codex app). It executes the recipe generated by this skill's
`flywheel-kickoff.sh`, then loops **poll Ôćĺ triage Ôćĺ act Ôćĺ journal** until the epic ships ÔÇö
respawning dead panes, nudging idle workers, unblocking env-blocked beads. Every launch
failure we have hit lives in its guard playbook (configÔćĺxhigh fallback, assignment gap,
dead panes, watch-command hangs).

**No supervisor/controller pane ÔÇö in either form.** A same-account Claude pane rate-limits
itself against your own session whether spawned fresh (`--cc=1`) or retrofitted
(`ntm controller`); recovery demands a second Anthropic account (conductor guard G1). The
conductor session *is* the supervisor.

**Before you spawn** (the conductor's Step 1 checks this too): Codex is current + logged
in ÔÇö a stale Codex **self-updates on first launch**, drops every pane to the shell, and
the init-prompt gets typed into zsh. Quick check: `echo ok | codex exec` returns text.

**Watch (human views):**
```bash
bv                              # bead progress dashboard (TUI, run in the repo)
ntm dashboard <name>            # agent liveness: token velocity, health, who's stalled
ntm attach <name>               # live panes
open http://127.0.0.1:8765/mail # messages + reservations (incl. the conductor lease)
```

**Teardown:** the conductor does it (its Step 5); manually `ntm kill <name> --force` then
`ntm cleanup`. Get back to a clean `main` once the PR merges (stash `.ntm/` stragglers).

---

## 4. Shipping

- **Commit + push per bead, immediately.** Each agent commits its own small change and pushes the moment a bead closes (unpushed = invisible to other agents; piling up changes creates the "mixed tree" that stalls commits). The lease guard checks reservations on commit.
- **Ship bead follows `.flywheel/profile` mode:** `solo` commits and pushes to the working branch with no PR. `team` opens the PR (`gh pr create`) and sets it **ready**, so CI + the preview env run, then **merge when green** ÔÇö directly for safe DX / non-prod / template changes, or with user approval otherwise. `gh pr merge <n> --auto --squash` merges the moment checks pass (or `--squash` once they're green). The agent should not block-poll CI; it opens ready, enables auto-merge when available, and lets CI/the human complete. Then **`/pr-closeout`**.
- **Verify:** **`/p-deploy-and-verify`**.
- **Close the loop:** `br changelog` Ôćĺ close the Linear issue. `ntm handoff create <name> --auto` for cross-session continuity.

Make shipping part of the swarm by encoding "open PR / deploy-verify / changelog" as final beads depending on the implementation beads.

---

## 5. No-worktrees enforcement

No built-in guard stops `git worktree add` (DCG allows it). Enforce by: **never** passing `--worktrees` to `ntm spawn`; the AGENTS.md flywheel-protocol line forbidding worktrees; and *(optional)* a custom DCG rule blocking `git worktree add`.
